# OpenClash 代理配置指南

## 概述

本文档说明如何配置 OpenClash 代理供 RSSHub 使用，以及多层机场架构的优势。

## 代理配置格式

### 无密码格式
```env
PROXY_URI=socks5h://192.168.1.1:7891
```

### 有密码格式
```env
PROXY_URI=socks5h://username:password@192.168.1.1:7891
```

### 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| 协议 | 推荐使用 `socks5h` | `socks5h://` |
| 用户名 | 代理认证用户名 | `myuser` |
| 密码 | 代理认证密码 | `mypass123` |
| IP | OpenWrt 路由器 IP | `192.168.1.1` |
| 端口 | OpenClash 代理端口 | `7891` |

## 端口说明

| 代理类型 | 默认端口 | 说明 |
|---------|---------|------|
| SOCKS5 | 7891 | 推荐，支持 DNS 远程解析 |
| HTTP | 7890 | HTTP 代理 |
| Mixed | 7890 | 混合端口 |

## 协议选择

| 协议 | DNS 解析 | 推荐度 |
|------|---------|-------|
| `socks5h` | 远程（通过代理） | ⭐⭐⭐ 推荐 |
| `socks5` | 本地 | ⭐⭐ |
| `http` | 本地 | ⭐ |

**推荐使用 `socks5h`**：DNS 查询通过代理服务器，防止 DNS 泄露。

## 密码特殊字符编码

如果密码包含特殊字符，需要进行 URL 编码：

| 特殊字符 | 编码后 |
|---------|-------|
| `@` | `%40` |
| `#` | `%23` |
| `:` | `%3A` |
| `/` | `%2F` |
| `?` | `%3F` |
| `&` | `%26` |

**示例**：密码 `pass@123` 应写成 `pass%40123`

## 多层机场架构

### 架构示例

```
┌─────────────────────────────────────────────┐
│              多层机场架构                     │
├─────────────────────────────────────────────┤
│  第一层：机场A + 机场B（2个）                 │
│      ↓ 掉线自动切换                          │
│  第二层：机场C + 机场D（2个）                 │
│      ↓ 掉线自动切换                          │
│  第三层：机场E（1个）                         │
└─────────────────────────────────────────────┘
```

### 优势

| 优势 | 说明 |
|------|------|
| **风险分散** | 5 个机场节点，被封概率分散 |
| **自动切换** | 某个节点被封，自动切换到下一个 |
| **高可用** | 三层保障，很难全部被封 |

### 被封风险分析

```
请求链路：
你的设备 → OpenClash → 机场节点 → 目标网站
(IP: A)    (路由器B)     (IP: C)    (Twitter等)

目标网站看到的 IP 是 C（机场节点IP）
OpenClash 设备 IP (B) 只是中转
你的真实 IP (A) 完全隐藏
```

**结论**：
- ✅ 你的真实 IP 安全
- ✅ OpenClash 路由器 IP 安全
- ⚠️ 机场节点 IP 可能被封（但有多节点备份）

## 查看端口方法

1. 登录 OpenWrt 管理页面
2. 进入 **服务 → OpenClash**
3. 查看 **覆写设置 → 常规设置**

## 故障排查

### 测试代理连接

```bash
# 在服务器上测试
curl -x socks5h://user:pass@192.168.1.1:7891 https://www.google.com
```

### 查看 RSSHub 日志

```bash
docker compose logs rsshub | grep -i proxy
```

### 常见问题

| 问题 | 解决方案 |
|------|---------|
| 连接超时 | 检查 IP 和端口是否正确 |
| 认证失败 | 检查用户名密码，注意特殊字符编码 |
| DNS 解析失败 | 使用 `socks5h` 而不是 `socks5` |
